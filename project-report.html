Project Overview

What is Hometown Hub?

A platform that solves the real problem of disconnection people face when they move away from their hometowns.

Problem Statement

People who migrate to different cities for work or education lose connection with their hometown communities. Existing social media platforms are too broad and do not provide a focused space for hyperlocal interaction, community updates, and local collaboration.

Our Solution

Hometown Hub creates a dedicated digital community space where individuals from the same city, village, or hometown can connect, communicate, organize events, and preserve local culture — no matter where they currently live.

Primary Objectives

Enable people from the same city or village to connect digitally. Create a centralized platform for community updates and discussions. Support local event organization and strengthen community bonding through participation.

Project Scope

Web-based responsive platform with community creation and membership, posts and discussions, event management, real-time notifications, user profiles with hometown details, and a comprehensive admin dashboard for platform moderation.

Frontend Layer

React SPA on Vercel CDN. AuthContext for state management, Axios for API calls, React Router for navigation and protected routes.

API Layer

Express REST API on Render. JWT middleware guards routes. Controllers handle all business logic. pg pool manages database connections.

Data Layer

Render-managed PostgreSQL with SSL. 8 relational tables, UUID primary keys, JSONB tags, and partial indexes for performance.

Database Design

PostgreSQL Schema

8 relational tables with UUID primary keys, foreign key constraints, and optimized indexes.

TablePrimary KeyKey ColumnsPurpose
usersUUIDemail, role, hometown, is_bannedUser accounts and profiles
communitiesUUIDslug, city, category, creator_idCommunity groups
membershipsCompositeuser_id, community_id, roleUser-community relationship
postsUUIDtype, tags (JSONB), is_pinned, is_deletedCommunity posts with soft delete
post_likesCompositepost_id, user_idPost engagement tracking
eventsUUIDstart_date, max_attendees, is_onlineCommunity events
event_attendeesCompositeevent_id, user_id, statusRSVP tracking
notificationsUUIDtype, is_read, recipient_idIn-app notification system

Key Design Decisions

UUID Keys — uuid-ossp extension generates all primary keys for distributed safety.
Soft Deletes — Posts use is_deleted flag; data preserved for moderation history.
JSONB Tags — Post tags stored as native PostgreSQL JSONB for flexible querying.
Partial Index — WHERE is_read = FALSE on notifications for fast unread badge queries.
Denormalized Counts — member_count, like_count stored to avoid expensive COUNT(*) joins.
SSL Support — Render PostgreSQL enforces SSL; handled via rejectUnauthorized: false.

Security

Security Implementation

Multiple layers of security protecting user data and platform integrity.

🔑

JWT Authentication

Signed tokens with 7-day expiry. Verified on every request via protect middleware before route handlers run.

🔒

Password Hashing

bcryptjs with 12 salt rounds. Plaintext passwords are never stored or returned in API responses.

🛡

Role-Based Access

protect, adminOnly, and optionalAuth middleware guards every route based on user role.

🌐

CORS Whitelist

Only registered frontend origins are allowed in production. All unknown origins are blocked.

✅

SQL Injection Safe

All database queries use parameterized placeholders. No string concatenation in queries.

📋

Input Validation

express-validator runs on all POST and PUT request bodies before any business logic executes.
